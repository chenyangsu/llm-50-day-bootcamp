The CUDA trap, and why the install is pinned

Your driver is 573.57, advertising CUDA 12.8. The GPU is a GTX 1650 Ti (Turing, sm_75).

Installing torch from PyPI the obvious way gets you a CUDA 13 build, which does not run on a 12.8 driver. It does not fail loudly either — it installs cleanly and then reports:

torch 2.13.0+cu130 | cuda available: False

which is the kind of thing you discover on day 25 when a sweep takes all afternoon on CPU. So the environment pins the CUDA 12.8 build explicitly:

uv pip install --python ~/.venvs/llm-bootcamp/bin/python \
  --index-url https://download.pytorch.org/whl/cu128 torch

If you ever rebuild the environment, use that line rather than a bare uv pip install torch.

Sanity check before any GPU day: python -c "import torch; print(torch.cuda.is_available())" must print True. If it prints False, you are about to train on CPU.

Rebuilding the environment from scratch

uv venv ~/.venvs/llm-bootcamp --python 3.12
uv pip install --python ~/.venvs/llm-bootcamp/bin/python \
  --index-url https://download.pytorch.org/whl/cu128 torch
uv pip install --python ~/.venvs/llm-bootcamp/bin/python \
  -r ~/Desktop/projects/llm-bootcamp-work/requirements.txt

The environment deliberately lives at ~/.venvs/llm-bootcamp, on the Linux filesystem, rather than beside the code under /mnt/c. WSL2’s bridge to the Windows drive is slow for operations over thousands of small files, which is exactly what a PyTorch venv is. Code and notebooks on /mnt/c are fine; the venv is not.

What your 4 GB of VRAM will and won’t do

This decides which days run on your laptop and which want Colab.

Weeks Locally? Notes
1–4 Comfortably micrograd and makemore barely touch the GPU. The day 22–26 GPT is about 10.8 M parameters at the video’s config and trains on TinyShakespeare in minutes. Day 25’s sweeps are the first thing that really benefits from the card.
5 Tight but workable A BERT-family encoder fine-tune fits with a modest batch size and mixed precision. Reach for gradient accumulation before reaching for Colab.
6 Depends Day 41’s LoRA run is where 4 GB starts choosing your model for you. Small model plus LoRA plus 4-bit fits; a full fine-tune of anything decoder-shaped probably doesn’t.
7 Embeddings yes, speedrun no Day 47’s retrieval work runs locally. Day 45’s nanochat speedrun needs 8×H100 — it is explicitly optional, and reading the pipeline carefully is the point.

When something doesn’t fit, Colab’s free T4 has 16 GB and covers everything in weeks 5–7.

Verify

If this prints a decreasing loss and cuda: True, you are ready for day 1:

import torch
print("cuda:", torch.cuda.is_available())

x = torch.randn(64, 3)
y = (x.sum(dim=1, keepdim=True) > 0).float()
w = torch.zeros(3, 1, requires_grad=True)
b = torch.zeros(1, requires_grad=True)

for step in range(200):
    loss = torch.nn.functional.binary_cross_entropy_with_logits(x @ w + b, y)
    loss.backward()
    with torch.no_grad():
        w -= 0.5 * w.grad; b -= 0.5 * b.grad
        w.grad = None; b.grad = None
    if step % 50 == 0:
        print(step, round(loss.item(), 4))

You will not fully understand loss.backward() yet. That is days 2 through 4.
